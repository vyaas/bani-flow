# READYOU.md — Carnatic knowledge graph: data schema and CLI reference

> Addressed to agents. All reads and writes go through the CLI. Never edit JSON files directly.

**Significance over completeness.** A musician belongs here only if they materially shaped the sound, transmission, or scholarship of the Carnatic tradition.

---

## Data files (current counts: 68 musicians · 49 edges · 156 ragas · 56 composers · 221 compositions · 12 recordings)

| path | role |
|---|---|
| `musicians/{id}.json` | one bare JSON object per musician |
| `musicians/_edges.json` | all guru→shishya edges (bare array) |
| `ragas/{id}.json` | one bare JSON object per raga |
| `compositions/{id}.json` | one bare JSON object per composition |
| `compositions/_composers.json` | all composers (bare array) |
| `recordings/{id}.json` | one bare JSON object per concert recording |
| `graph.json` | **derived** — regenerated by `render.py`; never edit directly |

Files prefixed with `_` are not treated as nodes or compositions. Each `{id}.json`'s `id` field must match its filename (without `.json`).

## CLI reads — `python3 carnatic/cli.py`

```bash
python3 carnatic/cli.py stats

# existence checks — exit 0 = found, exit 1 = not found
python3 carnatic/cli.py musician-exists    <id_or_label>
python3 carnatic/cli.py raga-exists        <id_or_name>
python3 carnatic/cli.py composition-exists <id_or_title>
python3 carnatic/cli.py recording-exists   <id_or_title>
python3 carnatic/cli.py url-exists         <youtube_url>

# lookups (add --json for full object)
python3 carnatic/cli.py get-musician       <id>
python3 carnatic/cli.py get-raga           <id>
python3 carnatic/cli.py get-composition    <id>
python3 carnatic/cli.py gurus-of           <musician_id>
python3 carnatic/cli.py shishyas-of        <musician_id>
python3 carnatic/cli.py lineage            <musician_id>
python3 carnatic/cli.py recordings-for     <musician_id>
python3 carnatic/cli.py compositions-in-raga <raga_id>
python3 carnatic/cli.py concerts-for       <musician_id>
python3 carnatic/cli.py co-performers-of   <musician_id>
python3 carnatic/cli.py concerts-with      <musician_id_a> <musician_id_b>
python3 carnatic/cli.py concert            <recording_id>  [--json]

# melakarta / cakra traversal
python3 carnatic/cli.py is-mela            <raga_id>
python3 carnatic/cli.py janyas-of          <mela_raga_id>
python3 carnatic/cli.py mela-of            <janya_raga_id>
python3 carnatic/cli.py cakra-of           <raga_id>
python3 carnatic/cli.py melas-in-cakra     <cakra_number>

python3 carnatic/cli.py validate
```

## CLI writes — `python3 carnatic/write_cli.py`

Every command validates referential integrity, writes atomically, and exits 0 on success/duplicate-skip or 1 on error. **After any write session, run `bani-render`.**

```bash
# ── Musicians ──────────────────────────────────────────────────────────────────

python3 carnatic/write_cli.py add-musician \
    --id <id> --label <label> --era <era> --instrument <inst> \
    --source-url <url> --source-label <label> --source-type <type> \
    [--born <year>] [--died <year>] [--bani <bani>]

python3 carnatic/write_cli.py patch-musician \
    --id <id> --field <field> --value <value>
# fields: label, born, died, era, instrument, bani  (id is immutable)

python3 carnatic/write_cli.py add-source \
    --musician-id <id> --url <url> --label <label> --type <type>

python3 carnatic/write_cli.py add-youtube \
    --musician-id <id> --url <yt_url> --label <label> \
    [--composition-id <id>] [--raga-id <id>] [--year <int>] [--version <text>]

# ── Edges ──────────────────────────────────────────────────────────────────────

python3 carnatic/write_cli.py add-edge \
    --source <guru_id> --target <shishya_id> \
    --confidence <float> --source-url <url> [--note <text>]

python3 carnatic/write_cli.py remove-edge \
    --source <guru_id> --target <shishya_id>

python3 carnatic/write_cli.py patch-edge \
    --source <guru_id> --target <shishya_id> \
    --field <field> --value <value>
# fields: confidence, source_url, note

# ── Ragas ──────────────────────────────────────────────────────────────────────

python3 carnatic/write_cli.py add-raga \
    --id <id> --name <name> \
    --source-url <url> --source-label <label> --source-type <type> \
    [--aliases <csv>] [--melakarta <int>] [--parent-raga <id>] [--notes <text>]

python3 carnatic/write_cli.py patch-raga \
    --id <raga_id> --field <field> --value <value>
# fields: name, parent_raga, melakarta, is_melakarta, cakra, notes
# is_melakarta: 'true'|'false'  · cakra/melakarta: integer or 'null'

# ── Composers & Compositions ───────────────────────────────────────────────────

python3 carnatic/write_cli.py add-composer \
    --id <id> --name <name> \
    --source-url <url> --source-label <label> --source-type <type> \
    [--musician-node-id <id>] [--born <year>] [--died <year>]

python3 carnatic/write_cli.py add-composition \
    --id <id> --title <title> \
    --composer-id <id> --raga-id <id> \
    [--tala <tala>] [--language <lang>] \
    [--source-url <url>] [--source-label <label>] [--source-type <type>] \
    [--notes <text>]
```

**Output prefixes:** `[NODE+]` `[EDGE+]` `[EDGE-]` `[EDGE~]` `[YOUTUBE+]` `[RAGA+]` `[RAGA~]` `[COMPOSER+]` `[COMP+]` `[SOURCE+]` · `SKIP (duplicate)` · `ERROR`

> **Recordings** and any operation not covered above: edit the relevant JSON file directly via a script or `apply_diff`, then run `bani-render`.

## Schema reference

### musicians/{id}.json

```json
{
  "id": "ramnad_krishnan",
  "label": "Ramnad Krishnan",
  "sources": [{ "url": "…", "label": "Wikipedia", "type": "wikipedia" }],
  "born": 1918,
  "died": 1973,
  "era": "disseminator",
  "instrument": "vocal",
  "bani": "semmangudi",
  "youtube": [
    {
      "url": "https://youtu.be/…",
      "label": "Composition · Raga · Tala — Artist",
      "composition_id": "nidhi_chala_sukhama",
      "raga_id": "kalyani",
      "year": 1965,
      "version": "Poonamallee 1965"
    }
  ]
}
```

| field | type | notes |
|---|---|---|
| `id` | string | snake_case, permanent |
| `label` | string | display name |
| `sources` | array | ≥1 source object; Wikipedia first |
| `born` / `died` | int \| null | year only |
| `era` | enum | `trinity` · `bridge` · `golden_age` · `disseminator` · `living_pillars` · `contemporary` |
| `instrument` | enum | `vocal` · `veena` · `violin` · `flute` · `mridangam` · `bharatanatyam` · `ghatam` · `other` |
| `bani` | string \| null | stylistic school / lineage label |
| `youtube` | array | may be `[]`; `composition_id`, `raga_id`, `year`, `version`, `performers` all optional |

**Source object:** `{ "url": "…", "label": "…", "type": "wikipedia"|"pdf"|"article"|"archive"|"other" }`

**`performers[]` (optional, ADR-070)**: a `youtube[]` entry may carry an explicit
performer roster — same `Performer` shape as `recordings/*.json` sessions
(`{ musician_id, role, unmatched_name? }`). When present, the host musician
(the parent node) **must** be listed; the validator enforces this. When absent,
the host is the implicit sole performer (back-compat — existing entries render
unchanged). Use this to credit accompanists on a single-track recording so they
flow through the graph the same way concert performers do. See
[recordings/READYOU.md](recordings/READYOU.md) for the shared `Performer` shape
and role vocabulary.

```jsonc
{
  "url": "https://youtu.be/M4J_HtniTQA",
  "label": "Gitarthamu · Surutti · Adi - MD Ramanathan",
  "composition_id": "gitarthamu",
  "raga_id": "surutti",
  "performers": [
    { "musician_id": "md_ramanathan",          "role": "vocal" },
    { "musician_id": "lalgudi_jayaraman",      "role": "violin" },
    { "musician_id": "umayalpuram_sivaraman",  "role": "mridangam" }
  ]
}
```

### musicians/_edges.json

```json
[
  {
    "source": "cs_sankarasivam",
    "target": "ramnad_krishnan",
    "confidence": 0.98,
    "source_url": "https://…",
    "note": "first guru; Muthiah Bhagavatar lineage stream"
  }
]
```

| field | type | notes |
|---|---|---|
| `source` | node id | the guru |
| `target` | node id | the shishya |
| `confidence` | float 0–1 | 0.95–1.0 = infobox / unambiguous; 0.85–0.94 = cross-confirmed; 0.70–0.84 = single source; <0.70 = speculative (must have `note`) |
| `source_url` | string | URL where the relationship is stated |
| `note` | string \| omit | qualifier on the nature of the relationship |

### ragas/{id}.json

```json
{
  "id": "kalyani",
  "name": "Kalyani",
  "aliases": ["Yaman (Hindustani equivalent)"],
  "melakarta": 65,
  "is_melakarta": true,
  "cakra": 11,
  "parent_raga": null,
  "sources": [{ "url": "https://en.wikipedia.org/wiki/Kalyani_(raga)", "label": "Wikipedia", "type": "wikipedia" }],
  "notes": "65th melakarta; all sharp notes except Sa and Pa"
}
```

| field | type | notes |
|---|---|---|
| `id` | string | snake_case, permanent |
| `name` | string | canonical name |
| `aliases` | array | alternate spellings; may be `[]` |
| `melakarta` | int \| null | 1–72 for mela ragas; `null` for janya |
| `is_melakarta` | bool | `true` = one of the 72 melakartas |
| `cakra` | int \| null | 1–12 for mela ragas; `null` for janya |
| `parent_raga` | string \| null | parent mela id for janya ragas; `null` for mela ragas |
| `sources` | array | ≥1 source object |
| `notes` | string \| null | musicological note |

**Cakra table:** 1 Indu (1–6) · 2 Netra (7–12) · 3 Agni (13–18) · 4 Veda (19–24) · 5 Bana (25–30) · 6 Rutu (31–36) · 7 Rishi (37–42) · 8 Vasu (43–48) · 9 Brahma (49–54) · 10 Disi (55–60) · 11 Rudra (61–66) · 12 Aditya (67–72)

### compositions/_composers.json

```json
[
  {
    "id": "patnam_subramanya_iyer",
    "name": "Patnam Subramanya Iyer",
    "musician_node_id": "patnam_subramanya_iyer",
    "born": 1845,
    "died": 1902,
    "sources": [{ "url": "https://en.wikipedia.org/wiki/Patnam_Subramanya_Iyer", "label": "Wikipedia", "type": "wikipedia" }]
  }
]
```

| field | type | notes |
|---|---|---|
| `id` | string | snake_case, permanent |
| `name` | string | canonical name |
| `musician_node_id` | string \| null | links to `musicians/{id}.json` if composer is also a graph node; otherwise `null` |
| `born` / `died` | int \| null | year only |
| `sources` | array | ≥1 source object |

### compositions/{id}.json

```json
{
  "id": "abhimana",
  "title": "Abhimana",
  "composer_id": "patnam_subramanya_iyer",
  "raga_id": "begada",
  "tala": "adi",
  "language": "Telugu",
  "sources": [{ "url": "https://www.karnatik.com/c2765.shtml", "label": "Karnatik.com", "type": "article" }],
  "notes": null
}
```

| field | type | notes |
|---|---|---|
| `id` | string | snake_case, permanent |
| `title` | string | canonical title |
| `composer_id` | string | references `_composers.json[].id` |
| `raga_id` | string | references `ragas/{id}.json` |
| `tala` | string \| null | `adi` · `rupakam` · `misra_capu` · `khanda_capu` · `tisra_triputa` · `ata` · `dhruva` · `other` |
| `language` | string \| null | `Telugu` · `Sanskrit` · `Tamil` · `Kannada` · `Malayalam` · `Other` |
| `sources` | array | may be `[]` if no external reference |
| `notes` | string \| null | musicological note |

### recordings/{id}.json

```json
{
  "id": "poonamallee_1965",
  "video_id": "_rj8fHJiSLA",
  "url": "https://youtu.be/_rj8fHJiSLA",
  "title": "Srinivasa Farms Concert, Poonamallee 1965",
  "short_title": "Poonamallee 1965",
  "date": "1965-01",
  "venue": "Srinivasa Farms, Poonamallee",
  "occasion": "Sangita Kalanidhi award celebration",
  "sources": [{ "url": "https://youtu.be/_rj8fHJiSLA", "label": "YouTube", "type": "other" }],
  "sessions": [
    {
      "session_index": 1,
      "performers": [
        { "musician_id": "ramnad_krishnan", "role": "vocal" },
        { "musician_id": null, "role": "violin", "unmatched_name": "V. Tyagarajan" }
      ],
      "performances": [
        {
          "performance_index": 1,
          "timestamp": "00:00",
          "offset_seconds": 0,
          "composition_id": "bale_balendu_bhushani",
          "raga_id": "reetigowla",
          "tala": "adi",
          "composer_id": "tyagaraja",
          "display_title": "bālē bālēndu bhūṣaṇi",
          "notes": null,
          "type": null
        }
      ]
    }
  ]
}
```

**Top-level fields**

| field | type | notes |
|---|---|---|
| `id` | string | snake_case, permanent, matches filename |
| `video_id` | string | 11-character YouTube video ID |
| `url` | string | full YouTube URL |
| `title` | string | human-readable title |
| `short_title` | string \| null | abbreviated title for UI |
| `date` | string \| null | ISO 8601 partial: `"1965-01"`, `"1967"`, `"1960s"` |
| `venue` | string \| null | physical location |
| `occasion` | string \| null | event context |
| `sources` | array | YouTube URL repeated here as source; `type` is always `"other"` |
| `sessions` | array | one per continuous performance block by a fixed ensemble |

**Performer object**

| field | type | notes |
|---|---|---|
| `musician_id` | string \| null | references `musicians/{id}.json`; `null` if unmatched |
| `role` | string | `vocal` · `violin` · `veena` · `flute` · `mridangam` · `ghatam` · `tampura` · `other` |
| `unmatched_name` | string | only present when `musician_id` is `null` |

**Performance object**

| field | type | notes |
|---|---|---|
| `performance_index` | int | 1-based within session |
| `timestamp` | string | `"MM:SS"` or `"HH:MM:SS"` |
| `offset_seconds` | int | same position as integer seconds |
| `composition_id` | string \| null | references `compositions/{id}.json` |
| `raga_id` | string \| null | references `ragas/{id}.json` |
| `tala` | string \| null | tala name |
| `composer_id` | string \| null | references `_composers.json[].id` |
| `display_title` | string \| null | transliterated title shown in UI |
| `notes` | string \| null | `"padam"`, `"javali"`, `"varnam"`, `"2 kalai"`, etc. |
| `type` | string \| null | only set to `"tani"` for percussion solos; omit otherwise |
| `version` | string \| null | distinguishes multiple recordings of the same composition by the same musician |

## Write workflow

```
1. python3 carnatic/cli.py stats                  # orient
2. existence checks before any write              # avoid duplicates
3. python3 carnatic/write_cli.py <cmd>            # atomic write
4. python3 carnatic/cli.py validate               # confirm integrity
5. bani-render                                    # rebuild graph.html
```

**Before adding any YouTube link:**
1. `url-exists <url>` — if found, stop.
2. `musician-exists` · `raga-exists` · `composition-exists` — add missing dependencies first.
3. `write_cli.py add-youtube …`

## Hard constraints

- **Never edit any `.json` file directly** — use `write_cli.py` or a script.
- **Never rename an `id`** in any file. IDs are permanent keys; filenames must match.
- **Never create a musician without ≥1 source** (Wikipedia preferred).
- **Never create a composition without verified `composer_id` and `raga_id`.**
- **Never infer an edge from shared `bani` alone.** Stylistic similarity ≠ lineage evidence.
- **Never silently drop an unmatched YouTube link or performer.** Flag with `musician_id: null` + `unmatched_name`.
- **Never add a speculative edge (`confidence` < 0.70) without a `note`.**
- **Never wrap any file** in an array or outer key — every file is a bare JSON object (or bare array for `_edges.json` and `_composers.json`).
- **Never set `musician_id` in a recording** to a value absent from `musicians/`.
- **Never edit `graph.json` directly.** It is regenerated by `bani-render`.
